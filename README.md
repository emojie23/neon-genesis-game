# 霓虹原罪 // NEON GENESIS

一款原创赛博朋克房间制双摇杆肉鸽。项目借鉴房间清理、资源门禁、遗物联动和多阶段首领等成熟设计原则，但角色、美术、名称、数值与关卡均为原创实现。

在线游玩：<https://emojie23.github.io/neon-genesis-game/>

## 运行

直接打开 `index.html` 即可游玩。若要启用离线缓存，建议在本目录启动静态服务器：

```powershell
python -m http.server 8080
```

浏览器访问 `http://127.0.0.1:8080/`。

## 操作

- `WASD`：移动
- `方向键`：四方向射击
- `Shift + ↑/→/↓/←`：切换 SERAPH / WRAITH / BEAST / VANGUARD
- 主键盘或数字小键盘 `1/2/3/4`：切换 VANGUARD / WRAITH / BEAST / SERAPH
- `E`：部署爆破电荷，可炸毁障碍和发现隐藏房
- `Space`：使用主动协议“熵骰”，重写当前房间遗物
- `Esc`：暂停
- 鼠标悬停：角色持续朝向指针；按住鼠标：向指针射击
- `M`：开关程序化背景音乐
- `F`：进入或退出全屏；手机端也可点击右上角全屏按钮
- 触屏：移动/射击双摇杆、炸弹、主动道具与形态按钮

## 当前内容

- 6 个手工楼层：每层均包含安全、战斗、免费遗物库、免费入场的黑市、隐藏与 Boss 房；第四层起追加深层特殊房，最终层还会出现升格实验室。
- 9 段连续开场记忆链，依次交代世界观、三种资源、四种武器形态、八类房间、三阶段 Boss 规则与六层下潜目标，支持跳过。
- 资源教学支持鼠标悬停、键盘聚焦和触屏点击；DATA、神经密钥、爆破电荷会弹出各自的动态说明卡。
- 房间协议拆分为八张并排卡片；悬停、键盘聚焦或触屏点击时，只突出当前房间并展示门禁、风险与收益。
- 启动、扫描、对向滑入、冲击、虹膜、地图构建、警戒与收束等差异化赛博转场，让每类教学信息拥有独立节奏。
- 4 种即时切换形态，共享生命与遗物，但拥有独立射速、伤害、弹速、敌人穿透、1–2 次墙面反弹和移动表现。
- 20 件核心遗物、6 类可叠加调校芯片与 3 类可重复载入的升格协议，支持多重射击、爆炸、连锁电击、敌人穿透、强化反弹、后射、自动无人机，以及伤害、射速、弹速、暴击、机动与护盾的持续成长。
- 核心遗物扩展为 20 件，并建立兵装、义体器官、禁忌契约、武器调校与升格协议五条视觉/机制谱系；新增第五残像、锯齿记忆、盲眼先知、低压圣龛、执刑标尺、无伤黑匣、机械合唱团和余烬收割器等原创构筑核心。
- 遗物不再只是抽象符号：祭台会根据家族与稀有度生成不同圣骸外框、扫描光柱与近距离信息卡；HUD 持续显示已装载遗物阵列，拾取横幅同时说明稀有度、家族、即时效果与推荐协同。
- DATA、神经密钥、爆破电荷、生命电池与防火墙护盾资源；HUD 中每 1 点生命对应 1 个独立电池格，低于约三分之一时显示静态低电量警示。
- 遗物库与黑市均不再消耗神经密钥：遗物库稳定提供二选一永久遗物，黑市免费进入并使用 DATA 购买三件固定库存。Boss 胜利后同样改为三选一，只增加决策宽度，不会一次拿走全部强化。
- 普通战斗房清理后固定掉落补给，并独立判定二至三选一发育终端；六层三个战斗房的终端概率依次为 `15/20/25% → 30/40/50% → 55/65/75% → 68/76/84% → 76/84/92% → 84/92/98%`。连续两房未出现终端时，下一房必定保底，让每局都有稳定构筑机会。
- 神经密钥初始为 0，仅在前三层的清房补给和早期幽灵缓存中产出，楼层基础权重为 `15% → 20% → 25%`，携带上限 3 枚；第四至第六层不再自然掉落钥匙，而是用早期储备开启消耗 1 枚的量子武库或最终层消耗 2 枚的升格实验室。
- 初始四形态基础火力整体收紧；前期终端以小型调校为主，中后期逐步提高武器模组和稀有协同权重。玩家从脆弱基础构筑出发，后期通过叠加芯片、遗物联动与深层房间奖励形成明显的火力质变。
- 熵骰主动道具，以房间净化次数充能。
- 赛博智天使、空相主母、六相母体及其三种深层升格体共 6 名首领；每名首领拥有三种形态、转阶段清弹与独立弹幕/走位逻辑。
- 隐藏墙爆破、门禁、房间奖励、Boss 遗物和楼层传送门。
- 多组连续霓虹长墙、斜向墙带、立柱与商店专属边缘地形；墙板会沿水平、垂直或斜向走势统一排列，炸开的缺口不会再被视觉连接线跨过；敌人出生点会主动避开墙体。
- 人物弹体使用连续碰撞检测，无法穿过室内墙体；守线者、赤兽与黄昏翼基础反弹 1 次，双生影基础反弹 2 次，幽灵协议可将反弹提升至上限 2 次。
- 小怪弹体会被室内墙体拦截；Boss 死亡时立即清除敌弹、危险区和残余召唤物。
- Boss 入场动画、三种战斗形态期间均会硬锁全部房门；异步入场信号绑定当前房间，异常丢失 Boss 时会自动重建战斗，击杀后即使离房再返回也会恢复传送门，避免空房软锁。
- 房门入口侧连续映射与短暂扫描转场，不会再传送到新房远端。
- 拾取使用玩家与掉落物的相对运动扫掠判定；进入磁吸范围后会补偿玩家速度并高速追赶，普通掉落约 0.08 秒、远距磁性掉落约 0.20 秒即可追上，不会拖在角色身后。
- 每次真正扣血后获得 1 秒无敌时间，连续碰撞与重叠弹幕不会瞬间重复结算。
- 受伤反馈采用短促轻震、中心透明的边缘警示和半透明无敌脉冲，不再用长时间随机震屏制造模糊感。
- 十八级威胁曲线按六层房间逐步提升：前三层从 `4/5/6 → 6/7/8 → 8/10/12` 起步，后三层继续增加敌群组合、远程单位、生命与射速；第一战仍只有 1 名射手并保留至少 1.55 秒暖机。普通房与 Boss 十八阶段均设置独立在场弹幕上限和换形喘息时间，后期有压力但开局不会铺满屏幕。
- 形态差异化暴击率、双倍暴击伤害、整数化大号高亮伤害数字与独立音画反馈。
- 四种形态拥有独立枪口爆闪、弹道尾迹、命中几何特效、预生成霓虹光片和纯视觉后坐力；赤兽与黄昏翼附带轻微射击震屏。光片替代逐弹实时模糊，高密度射击仍能保持清晰特效与流畅帧率。
- 玩家弹使用白芯形态色尾迹，小怪弹使用暗芯方形能量体，Boss 弹使用旋转六边形与黄色警戒双环。
- Canvas 美术升级为原创“赛博宗教遗骸”语言：四种角色形态拥有独立面罩、武器、翼片、脊刺、盾甲和残像轮廓；五类小怪改为爬行义肢、浮游眼炮、冲锋兽颚、机械圣坛与折翼无人机；三类 Boss 分别使用破损光环、预测母眼与六瓣原罪体结构，并随阶段继续增生。
- 六层房间加入装甲拼板、螺栓、血色记忆污迹与跨场神经缆线；遗物、角色、Boss 与房间图案共用家族化色彩和轮廓规则，减少通用霓虹几何块的重复感。
- 房间掉落拥有放大弹出、双层扩散环和高亮边缘，重要资源更容易被注意到。
- 112 BPM 原创程序化合成配乐，重构后的可听频段声部与动态压限总线会在 Boss 战增强节奏；四种武器形态各有独立的缓存式射击音色。音频恢复请求与射击声部均有硬上限，浏览器挂起音频时不会再堆积节点；未成功启动时按钮会显示 `RETRY`。
- HUD 与小地图只在状态变化时刷新，避免持续射击期间反复重建 DOM。
- 桌面与横屏触控布局、Web Audio、震动、屏幕震动与离线缓存。
- 手机竖屏会显示明确的旋转引导；转为横屏后按动态可视视口、刘海安全区与短屏高度自动收缩，右上角支持网页全屏并尝试锁定横向，iPhone 不支持网页全屏时会提示添加到主屏幕。

## 自动化验证

`tests/smoke.cjs` 使用 Playwright 检查九段教学、三资源交互、八类房间说明、免费遗物库/黑市、Boss 换形喘息、受伤反馈、音频、武器特效、穿墙与反弹、掉落拾取、六层推进及最终胜利。`tests/progression.cjs` 专门验证六层成长概率、连续落空保底、前三层钥匙经济、深层门禁、二/三选一互斥、商店库存持久化、隐藏房防重复刷取、奖励池耗尽升格兜底、前弱后强输出曲线与六名 Boss 完整流程。`tests/art-relics.cjs` 检查 20 件遗物的家族/稀有度/协同信息、遗物阵列、第五残像、暴击穿透、低压增幅、反弹/Boss 乘区、无伤奖励、双无人机与新版 Canvas 色彩复杂度；`tests/boss-room-state.cjs` 会验证 Boss 入场定时器归属、三形态真实撞门锁定、异常丢失后的战斗恢复、击杀后传送门持久化与软锁自愈；`tests/difficulty.cjs` 会实际遍历十八个战斗房与 Boss 十八阶段，压测敌弹和召唤数量上限；`tests/mobile.cjs` 回归触控布局；`tests/performance.cjs` 验证基础射击、满配双生影及真实战斗墙体场景的帧时间预算。测试模式仅在 URL 包含 `?test=1` 时开放调试接口。
